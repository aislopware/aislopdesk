## Network / transport
- ‚úÖ **Assume a trusted WireGuard mesh (e.g. NetBird/Tailscale), direct P2P.** Relay = degraded fallback: surface it + warn, do NOT engineer a workaround. ‚Üí [13]
- ‚úÖ **MEASURED (2 real machines, M1 Max ‚Üî M2 Pro, over a WireGuard mesh):** RTT **avg 11ms** (8.5‚Äì14.5), 0% loss, **direct P2P OVER THE INTERNET** (NAT hole-punch, local 192.168 ‚Üî remote public IP, not same-LAN). Validates the "5‚Äì20ms direct P2P" assumption. ‚Üí [18 ¬ß0]
- ‚úÖ **No app-layer encryption** ‚Äî the mesh provides WireGuard E2E + deny-by-default per-port ACLs. ‚Üí [13]
- ‚úÖ **Terminal = plain TCP** (reliable; escape sequences may split across reads ‚Üí only buffering is needed, no loss-recovery). ‚Üí [13], [12]
- ‚úÖ **`TCP_NODELAY` mandatory** on every PATH 1 socket right after connect ‚Äî Nagle coalescing 1-character writes can add +200ms/keystroke (high impact, one setsockopt line). ‚Üí [17]
- ‚úÖ **Dual data/control channel** (PTY bytes ‚Äñ `TIOCSWINSZ` resize+intent) ‚Äî burst output doesn't delay the resize-ack (Zellij lesson). ‚Üí [17]
- ‚úÖ **GUI video = plain UDP** ‚Äî drop QUIC (TLS is redundant on top of the WireGuard tunnel). ‚Üí [03], [13]
- ‚úÖ **Do NOT pin `requiredInterfaceType`** (a userspace-WG interface shows up as `.other` ‚Üí pinning would break it). ‚Üí [13]
- ‚úÖ **serviceClass/DSCP has no effect through the tunnel** (WireGuard zeroes DSCP) ‚Üí app-layer adaptive rate. ‚Üí [13]
- ‚úÖ **Discovery:** mDNS/Bonjour is same-LAN only (does not traverse the mesh) ‚Üí connect by mesh DNS/IP. ‚Üí [13]

## FPS / latency

> **RE-SCOPE 2026-06-16 ‚Äî latency-first coding reframe.** Aislopdesk is a coding tool, NOT a game/video stream. After a long push chasing Parsec-class motion smoothness, the priorities are reset, in strict order: **(P1) zero input latency** (keyboard + mouse, incl. the visual echo), **(P2) sharp-when-stable** (transient blur during scroll/motion is fine; the instant motion stops, text must re-sharpen fast), **(P3) fps 23‚Äì30 is enough** (may drop lower on bursts). Motion smoothness and heavy loss-recovery of transient frames are now low priority and may be simplified. The bullets below supersede the earlier "smoothness-first" decisions.

- ‚úÖ **GUI video-path defaults to 30 fps with idle-skip** (`--fps` knob; 60 still reachable). Reverted toward the original ~24‚Äì30 target: 30 fps halves the per-frame byte budget at the same ceiling ‚Üí crisper individual frames + less self-induced congestion/loss, and transient scroll blur is acceptable. Capture stays at 2√ó the encode fps (~60 Hz) so a typed-character change is still picked up within ~16 ms, not quantized to the 33 ms encode slot. Idle-skip keeps a static screen near-zero. ‚Üí [12], [09]
- ‚úÖ **Client presentation defaults to PRESENT-ON-ARRIVAL** (jitter depth 1, no playout hold). The deadline pacer + adaptive playout buffer (HW-validated for Parsec-style smoothness: present-gaps 0.37%‚Üí0%, max-hold 258‚Üí91 ms) add a standing buffer to the keypress‚Üíecho loop, and P1 input latency now outranks smoothness. The adaptive 1‚Üî2 depth boost (`pacer_depth_policy`) is also default-off (depth pinned at 1). `AISLOPDESK_PACER=deadline` (+ `AISLOPDESK_ADAPTIVE_DEPTH=1`) restores the smoothness-tuned pacer for a jittery-WAN A/B. ‚Üí [17], [11]
- ‚úÖ **Sharp-on-stable: crisp re-anchor ~300 ms after motion stops** (`StaticIDRDecider`, quiet window 1.0 s‚Üí300 ms, poll tick 0.25 s‚Üí80 ms; `AISLOPDESK_QUIET_MS` / `AISLOPDESK_IDR_TICK_MS`). The in-motion path stays cheap/blurry (pure-VBR + QP ceiling); the QP18 crisp IDR rides the same session (no client decoder rebuild). ‚Üí [17]
- ‚úÖ **HiDPI 2√ó virtual-display capture is default-ON** (`--no-virtual-display` / `AISLOPDESK_VD=0` to disable) ‚Äî the single biggest text-sharpness lever; a 1√ó fallback now logs a loud warning rather than silently shipping soft text. ‚Üí [12]
- ‚úÖ **Terminal-path latency = network RTT** (~1‚Äì5ms LAN-direct), no vsync/encode/decode. ‚Üí [13]
- ‚úÖ **The floor-analysis [11] applies to the GUI video path only.** Chasing motion-to-photon <16ms / 120fps / ProMotion / beam-racing is over-engineering for coding and stays out of scope; the goal is **zero felt input latency + crisp-on-stable at ~30 fps**, not motion smoothness or a hard latency floor. ‚Üí [11], [12]

## Single language: Swift + native SIMD kernels  *(was: Rust core / FFI boundary)*
- ‚úÖ **REVERSAL (2026-06-19): one language = Swift. The Rust core (`aislopdesk-core`) + the FFI boundary (`aislopdesk-ffi`, cbindgen, generated header) are being removed; every codec/FEC/controller/terminal-protocol module is reabsorbed into optimized native Swift.** Rationale: the two-language cost was the **boundary, not the languages** ‚Äî ~21K LOC of FFI machinery (ffi crate ~15.9K + generated header ~5.2K) bridged to only ~700 LOC that genuinely needs native SIMD. FFI is a *measured tax* (bulk per-pixel codecs are 5‚Äì7√ó slower over it, so they were already kept native Swift); controller/codec logic lowers through LLVM identically in either language, so it is perf-neutral. Collapsing to Swift **removes the marshalling tax** (per-frame boundary crossings get *faster*) and **shrinks total `unsafe`** (the tree's only `unsafe` was the deleted ffi crate). ‚Üí reverses the three SUPERSEDED entries below.
- ‚úÖ **The ~700 LOC of genuine SIMD ‚Äî RS GF(2‚Å∏) region-multiply + frame-hash, NEON split-table `vqtbl1q_u8` ‚Äî stays as native code in a tiny C SwiftPM target Swift links directly** (bridging header, **no cbindgen / no marshalling** ‚Äî C-interop is free, unlike the Rust-FFI ABI tax). Same `tbl.16b` codegen as the Rust NEON ‚áí perf-identical. Pure-Swift `SIMD16<UInt8>` is **rejected for GF-multiply** (no `vqtbl` table-lookup exposed ‚Üí slower). This C target becomes the **only** remaining `unsafe`; allowlist it and SwiftLint-ban `Unsafe*` elsewhere to approximate the lost compiler-proven `#![forbid(unsafe_code)]` with a CI-enforced convention.
- ‚úÖ **`golden_vectors.json` becomes a single-impl Swift regression pin** (was cross-language Swift‚ÜîRust `golden_parity`). It was generated FROM the original Swift codecs and the Rust port is bit-identical, so each module reabsorbs to Swift pinned by the **same** vectors. Bit-exactness rules are unchanged and load-bearing: float math stays **separate `mul`+`add`, never fma**; wire stays **manual big-endian binary** (no JSON/Codable on the hot path); decoders **validate-then-drop** corrupt UDP (return optional/throw, never force-unwrap untrusted input); C-struct bools are `u8` read `!= 0`.

## Coding-workspace redesign (2026-06-20) ‚Äî binding; plan in [42](42-implementation-plan.md), research in [41](41-redesign-research.md)
- ‚úÖ **Retire the infinite canvas ‚Üí `Session ‚Üí Tab ‚Üí Pane` n-ary tiled split tree.** The free-floating `Canvas` (drag/snap/non-overlap/camera) is the wrong primitive for a coding tool; every competitor (tmux/Zellij/WezTerm/Muxy/Herdr/Warp) converged on a recursive split tree under named sessions/tabs because it is keyboard-drivable, deterministic, and trivially serializable. Arity = **n-ary** (Zellij `TiledPaneLayout`): closing child N redistributes flex equally among siblings instead of WezTerm's "sibling eats all freed space". The split tree stores only `PaneID`s; `PaneSpec` lives in a `Session.specs` side table so a rename never churns a tree diff. **`WorkspaceStore`'s intent-tree ‚Üí `reconcile()` ‚Üí registry pattern is preserved verbatim** ‚Äî only the leaf-id *source* moves (`canvas.allIDs()` ‚Üí `workspace.allPaneIDs()`); the invariant `Set(registry.keys) == Set(allPaneIDs())` is unchanged.
