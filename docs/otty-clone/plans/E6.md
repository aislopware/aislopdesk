## Design decisions (where otty cannot map 1:1 onto the remote architecture)

1. **Per-pane rows vs per-tab rows.** otty's sidebar is one row per *tab*; our rail is V1 "Panes"
   granularity (one row per visible pane of the active session — `RailRowsBuilder.rows`). For the common
   single-pane tab this is 1:1. The `#N` badge is the **tab** shortcut number (the `⌘1…⌘9` target =
   tab index+1), so split-tab panes share the same `#N`. We keep per-pane granularity (it is the
   shipped model) and attach the tab-derived `#N` to each row. Faithful for the dominant case; stated
   as the deliberate mapping.

2. **Created/Updated timestamps are EPHEMERAL (no schema bump, no migration).** otty's "Created Time"
   = oldest-opened first; our `session.tabs` array is already in insertion order (`newTab` appends), and
   persistence preserves array order — so **Created sort == array order**, no timestamp needed.
   "Updated Time" needs live recency: we keep a **runtime** `tabLastActiveAt: [TabID: Date]` mirror on
   the store (NOT persisted, NOT on the `Tab` Codable struct). This avoids a `Tab`/`Session` schema bump
   and any v11→v12 migration (honoring the no-backcompat directive — nothing to migrate). On relaunch
   recency resets to load order, which is acceptable (otty's updated-time tracks live activity too).

3. **By-Project key source.** A true `git rev-parse --show-toplevel` is a **host** fact (the terminal is
   remote — the client has no local FS). The faithful equivalent reuses E4's `gitStatus` RPC. To keep
   E6 self-contained and de-risked, the pure ordering engine takes a `projectKey: (PaneID) -> String?`
   **closure**; the store implements it preferring a cached git toplevel and **falling back to
   `lastKnownCwd`**. So By-Project works immediately on cwd (WI-3), and WI-7 upgrades the key to the real
   git toplevel (adds `repoRoot` to the E4 payload + eager debounced fetch when grouping == byProject).
   If WI-7 is cut, By-Project still groups by cwd — never a hard dependency.

4. **Manual drag-reorder must not tear down sessions.** `moveTab` only permutes the `session.tabs`
   array; the leaf set is unchanged so `reconcileTree()` is a registry no-op (no `teardown`, no surface
   rebuild — memory rule "never tear down surface"). Manual sort = render rows in `session.tabs` order;
   the drag updates that array.

5. **Badge fusion priority.** One badge per row, resolved by a pure `TabBadgeResolver` with a fixed
   precedence (most-urgent wins), distilled from `progress-state.md` + `parallel-tasks.md`:
   `awaitingInput (agent needsPermission / hand)` > `error (completion .failure / agent error)` >
   `running (isShellBusy / agent working — spinner)` > `sudo (shield)` > `caffeinate (coffee)` >
   `completed/finished (completion .success — checkmark→accent dot)` > `none`. (Caffeinate/sudo sit
   below active states so a running privileged command still spins; they surface when the shell is at
   rest, matching otty's "session is active" semantics.)

### WI-3 — Tab grouping/sort preference + pure `TabOrderingEngine` + `moveTab` (WorkspaceCore, headless)

**New file:** `Sources/AislopdeskWorkspaceCore/Workspace/Tabs/TabOrdering.swift`
- `public enum TabGrouping: String, Codable, Sendable, CaseIterable { case none, byProject, byDate }`
- `public enum TabSort: String, Codable, Sendable, CaseIterable { case created, updated, manual }`
- `public struct OrderedTabGroup: Equatable, Sendable { let header: String?; let tabIDs: [TabID] }`
  (`header == nil` ⇒ ungrouped flat list, no section chrome).
- `public enum TabOrderingEngine` with one pure static:
  ```
  static func groups(tabs: [Tab],
                     grouping: TabGrouping,
                     sort: TabSort,
                     projectKey: (PaneID) -> String?,
                     lastActiveAt: (TabID) -> Date?,
                     now: Date) -> [OrderedTabGroup]
  ```
  - **sort within a group:** `.created`/`.manual` ⇒ preserve incoming `tabs` array order; `.updated` ⇒
    stable-sort by `lastActiveAt` descending (nil last). Use an **ordered** comparison (stable sort by a
    key), never a bare `<` ternary on a possibly-equal key — ties preserve array order.
  - **grouping:** `.none` ⇒ one `OrderedTabGroup(header: nil, …)`. `.byProject` ⇒ bucket by
    `projectKey(tab.activePane)` (nil ⇒ an "Other" bucket); group headers = the project's last path
    component; groups ordered by first-appearance in `tabs`. `.byDate` ⇒ bucket by `lastActiveAt` into
    Today / Yesterday / Earlier (compute with `Calendar`; pure given `now`).
  - Pure, deterministic, no SwiftUI, no clock read (caller passes `now`) — fully headless-testable.

**New file (pure op):** `Sources/AislopdeskWorkspaceCore/Workspace/Domain/Tree/WorkspaceTreeOps+MoveTab.swift`
(or add to `WorkspaceTreeOps.swift`)
- `static func moveTab(from: Int, to: Int, in ws: TreeWorkspace) -> TreeWorkspace` — permute the active
  session's `tabs` array (clamp indices; keep `activeTabIndex` pointing at the SAME tab id after the

**Edit:** `Sources/AislopdeskWorkspaceCore/Workspace/Store/WorkspaceStore.swift`

- Public mutators `setTabGrouping(_:)` / `setTabSort(_:)` (idempotent) that **persist** through the
